## 一、战略定位

### 1.1 一句话定位

> **XTCRM 是中国制造业中小企业的 AI 驱动客户关系管理系统。开源、私有部署、不按人头收费。**

### 1.2 目标客户画像

| 维度 | 核心圈（制造业工厂） | 第二圈（贸易/批发） | 第三圈（工程/项目） |
|------|---------------------|---------------------|---------------------|
| **典型企业** | 五金加工、钣金、电子元器件、工业皮带、化工、陶瓷、模具 | 工业品贸易商、建材批发、设备代理商、原料经销商 | 设备安装、工程承包、弱电工程、暖通工程 |
| **团队规模** | 3-30 人销售团队 | 5-20 人销售团队 | 5-50 人项目团队 |
| **客单价** | 5 万 - 500 万 | 1 万 - 50 万 | 20 万 - 2000 万 |
| **销售周期** | 2 周 - 6 个月 | 1 天 - 2 周 | 3 个月 - 2 年 |
| **核心痛点** | 报价跟进靠记忆、销售离职客户丢、复购周期无人盯 | 客户多记不住、订单频容易漏、价格变动通知不及时 | 项目节点多、多人协作乱、标书方案重复写 |
| **决策人** | 老板本人（40-55 岁） | 老板或销售总监 | 老板或项目经理 |
| **IT 能力** | 会装宝塔面板 | 会装宝塔面板 | 可能有兼职 IT |

### 1.3 核心差异化（不可替代性）

```
市场上每出现 100 个通用 CRM，就有一个在问：
"为什么我的 CRM 不知道这个客户该回购了？"
"为什么我还要自己去翻报价记录？"
"为什么系统不能帮我写跟进的微信？"

XTCRM 的回答：
┌─────────────────────────────────────────────┐
│  ① AI 洞察引擎 — 系统主动找人，不是人找系统   │
│  ② 内容引擎   — CRM 数据直接变成营销内容     │
│  ③ 开源免费   — 不按人头收费，数据在自己手里  │
│  ④ 技能包     — 报价/发票/内容按需安装       │
└─────────────────────────────────────────────┘
```

### 1.4 不做什么（同等重要）

| 不做的 | 原因 |
|--------|------|
| ☒ 零售/电商 CRM | 客户量级不同（日增几百 vs 月增几十），洞察引擎规则完全失效 |
| ☒ SaaS 多租户云服务 | 与"数据在自己服务器"的信任承诺矛盾 |
| ☒ 移动端原生 App | 维护成本高，PWA/响应式网页可覆盖 90% 场景 |
| ☒ 财务/ERP 深度集成 | 那不是 CRM 的事。做轻量对接即可（导出/导入） |
| ☒ 国际化/多语言 | 中国市场足够大，先把中文做到极致 |
| ☒ AI 替代销售做决策 | AI 是建议者，不是决策者。决策权永远在人手里 |

## 二、产品愿景

### 2.1 三年愿景（2028）

一个制造业老板早上打开 XTCRM，他看到的是：

```
┌──────────────────────────────────────────────────┐
│  早上好，张总。                                    │
│                                                   │
│  🔴 需要你关注（3）                                │
│  • 深圳钜达电子 报价已发 12 天无回应，建议今日跟进     │
│  • 东莞力恒五金 采购周期偏离 40%，疑似流失           │
│  • 销售王磊 本周跟进次数低于团队均值 60%             │
│                                                   │
│  🟡 机会提醒（2）                                  │
│  • 惠州鑫达 历史 3 月采购旺季，建议提前联系          │
│  • 佛山宏业 刚通过了一项你公司主营产品的预算         │
│                                                   │
│  📊 本周概览                                       │
│  新客 +5  报价 12  成交 3  回款 48万               │
│  排名：李华第一（成交 2 单/28 万）                  │
└──────────────────────────────────────────────────┘
```

**他不需要点任何菜单。** 系统已经把今天该知道的事、该做的事，全部放在他眼前。

### 2.2 产品北极星指标

> **"每周至少 5 天打开 XTCRM 的用户占比"**

不是注册数，不是下载量，不是 Star 数。  
如果用户每天都打开，说明系统真的在帮他赚钱。  
如果用户装了就吃灰，说明系统还是个电子台账。

## 三、开发原则

### 3.1 十大产品原则

| # | 原则 | 说明 | 反面教材 |
|---|------|------|----------|
| 1 | **主动优于被动** | 系统推送给用户，不是等用户来查 | 传统 CRM 的报表中心 |
| 2 | **默认够用，配置可选** | 出厂就能用，高级参数会配再配 | 一上来就问 20 个设置项 |
| 3 | **AI 是副驾驶，不是自动驾驶** | 给建议，给理由，让人做决定 | AI 自动关闭订单、自动发报价 |
| 4 | **一个场景一个入口** | 不要为了"统一"把不同的事塞进同一个页面 | 把客户、订单、合同全堆在详情页 |
| 5 | **10 秒看到结果** | 任何操作 10 秒内必须有反馈，包括 AI 调用 | AI 生成等 2 分钟，用户以为卡死了 |
| 6 | **数据主权归用户** | 一键导出所有数据，格式标准、完整、可用 | 导出 CSV 乱码、附件丢失 |
| 7 | **性能不妥协** | 1000 客户 + 5000 跟进记录，列表页加载 < 1s | 翻个页等 3 秒 |
| 8 | **移动端可用** | 销售外勤时不打开电脑也能写跟进、查客户 | 手机打开是 PC 版缩小到看不清 |
| 9 | **技能包即插即用** | 安装一个技能包 ≤ 3 步：复制文件 → 执行迁移 → 刷新页面 | 装个插件要改 5 个配置文件 |
| 10 | **永不静默失败** | 任何出错都给明确提示 + 建议操作 | "操作失败"四个字，用户一脸懵 |

### 3.2 技术决策原则

| 场景 | 决策 |
|------|------|
| 用现有轮子 vs 自己造 | 优先用成熟的（DeepSeek、WangEditor、Ant Design），核心差异化的（洞察引擎、SkillPipeline）自己造 |
| 性能 vs 开发速度 | 用户能感知的（列表加载、AI 响应）优化性能。内部工具（迁移脚本、定时任务）够用就行 |
| 通用设计 vs 行业定制 | 核心层通用（客户/报价/订单），应用层行业化（洞察规则参数、内容引擎模板） |
| 向前兼容 vs 推倒重来 | 数据库只加列不改列，API 只加参数不改返回格式。V1 前端可维护但不再加新功能 |

## 四、三阶段路线图

### 第一阶段：根据地（2026 Q3-Q4）

> **目标：让制造业老板"用了就离不开"。**  
> **核心指标：活跃用户周打开率 > 70%，GitHub Stars > 500。**

#### Q3（7-9 月）：夯实核心体验

| 优先级 | 功能 | 类型 | 说明 | 预估工时 |
|--------|------|------|------|----------|
| **P0** | 洞察引擎 V2 | 后端 | 5 条规则增加行业参数模板（制造业/贸易/工程三套默认值），支持用户自定义阈值 | 2 周 |
| **P0** | 洞察卡片重设计 | 前端 | 从文字列表升级为卡片式，优先级颜色标记（红/橙/蓝），一键指派给销售 | 1.5 周 |
| **P0** | 跟进时间线增强 | 前后端 | 时间线增加 AI 建议入口，每条跟进记录旁边显示"生成跟进建议"按钮 | 1 周 |
| **P1** | 移动端跟进 | 前端 | 响应式适配：写跟进记录、查客户信息、接收洞察推送。不做原生 App，走 PWA | 2 周 |
| **P1** | 客户 360° 视图 | 前端 | 客户详情页整合：基本信息 + 联系人 + 跟进时间线 + 报价/订单/合同列表 + AI 画像，一个页面看完 | 2 周 |
| **P1** | 智能搜索 V2 | 后端 | 搜索支持自然语言（"深圳做五金的客户""上个月没联系的"），对接 AgentService 已有工具 | 1 周 |
| **P2** | 演示数据包 | 数据 | 内置一套五金加工行业的演示数据（50 客户 + 200 跟进 + 30 报价 + 15 订单），新用户一键导入 | 1 周 |
| **P2** | 新手引导 | 前端 | 首次登录 5 步引导：导客户 → 看洞察 → 写跟进 → 建报价 → 看仪表盘 | 1 周 |

#### Q4（10-12 月）：打造"一见钟情"体验

| 优先级 | 功能 | 类型 | 说明 | 预估工时 |
|--------|------|------|------|----------|
| **P0** | 仪表盘 V2 | 前后端 | 首页重做：洞察卡片（最上方）+ 待办列表 + 漏斗图 + 团队排名。**去掉"欢迎页面"，登录即看数据** | 2 周 |
| **P0** | 通知中心 | 前后端 | 站内通知 + 浏览器推送 + 微信模板消息（可选），洞察警报实时触达。基于已有 NotificationService 增强 | 1.5 周 |
| **P0** | 数据导出 | 后端 | 一键全量导出：客户 + 跟进 + 报价 + 订单 + 合同，JSON/CSV 双格式，附件打包 zip。**这是信任基建** | 1 周 |
| **P1** | 行业模板系统 | 后端 | 制造业/贸易/工程三套行业模板：预设字段、洞察规则参数、仪表盘布局、内容引擎语料。新建租户时选择 | 2 周 |
| **P1** | 报价转化率看板 | 前端 | 报价-成交漏斗（按人/按时间），报价超期排行榜，丢单原因统计 | 1.5 周 |
| **P1** | 采购周期分析 | 后端 | 基于订单历史，自动计算每个客户的采购周期（均值/标准差/趋势），替代当前硬编码阈值 | 1 周 |
| **P2** | 团队协作增强 | 前后端 | 客户共享支持"可编辑/只读"权限，协作动态在时间线显示，@提及同事 | 1.5 周 |
| **P2** | 部署脚本 | 运维 | 一键部署脚本（Docker Compose），含 Nginx + PHP 7.4 + MySQL + Redis。**把"30 分钟部署"缩短到 5 分钟** | 1 周 |

#### 第一阶段里程碑验收标准

- [ ] 制造业老板首次登录 → 导入演示数据 → 看到洞察卡片 → 写第一条跟进，全程 ≤ 10 分钟
- [ ] 每日洞察卡片准确率 > 80%（用户不点"忽略"）
- [ ] 列表页 1000 客户 + 5000 跟进记录，加载时间 < 1s
- [ ] 手机端写跟进、查客户可用（不需要缩放/横屏）
- [ ] GitHub README 含 5 分钟部署视频 + 10 分钟产品演示视频
- [ ] 文档站点上线（VitePress）：部署指南 + 用户手册 + 开发文档

### 第二阶段：立标杆（2027 Q1-Q2）

> **目标：在制造业 CRM 赛道建立口碑，成为"制造业老板推荐给同行"的产品。**  
> **核心指标：GitHub Stars > 2000，自然流量注册 > 500/月。**

#### Q1（1-3 月）：AI 深度化

| 优先级 | 功能 | 类型 | 说明 | 预估工时 |
|--------|------|------|------|----------|
| **P0** | 洞察引擎 V3 | 后端 | 引入 AI 增强规则（B 类）：流失风险客户自动生成挽回策略、回购窗口客户自动生成推荐产品清单。规则数量从 5 条扩展到 8 条 | 3 周 |
| **P0** | AI 客户健康分 | 后端 | 每个客户自动计算健康分（0-100），维度：活跃度/复购规律/回款及时度/投诉次数。Dashboard 显示健康分趋势 | 2 周 |
| **P1** | AI 周报 | 后端 | 每周一自动生成团队周报：本周关键数字 + 异常客户 + AI 建议。推送到通知中心 + 支持复制到微信 | 1.5 周 |
| **P1** | 内容引擎 V2 | 前后端 | 选题扫描增加行业热词追踪（抓取行业招标/展会信息），生成内容支持插入产品参数表、报价区间 | 2 周 |
| **P1** | 对话助手增强 | 后端 | 对话增加上下文记忆（同一天内的对话记住前面聊的内容），支持"上次那个客户的报价再发我一下" | 1.5 周 |
| **P2** | AI 训练反馈 | 后端 | 每个 AI 建议增加 👍/👎 反馈，数据回写到 AiSuggestion 表，用于后续 prompt 优化 | 1 周 |

#### Q2（4-6 月）：行业纵深

| 优先级 | 功能 | 类型 | 说明 | 预估工时 |
|--------|------|------|------|----------|
| **P0** | 报价管理增强 | 前后端 | 报价对比表（同一客户多版本报价）、报价有效期倒计时、批量调价（按产品/按客户分组） | 2 周 |
| **P0** | 订单交期看板 | 前端 | 以时间线/甘特图展示所有订单的生产/交付进度。逾期订单红色高亮。**这是制造业刚需** | 2 周 |
| **P1** | 产品推荐引擎 | 后端 | 基于客户历史订单 + 同类客户购买记录，AI 推荐未购买但可能有需求的产品 | 1.5 周 |
| **P1** | 售后工单技能包 | 技能包 | 设备销售后的安装/维修/巡检管理。工单状态机、服务记录、客户签字确认 | 3 周 |
| **P1** | 标书/方案生成 | 技能包 | 输入客户需求 → AI 生成技术方案草稿（含公司介绍、产品参数、案例、报价概算） | 2 周 |
| **P2** | 合同模板引擎 | 后端 | 合同模板支持变量替换（客户名/金额/日期/产品清单），从订单数据自动填充。对标 SkillTemplate | 1.5 周 |

#### 第二阶段里程碑验收标准

- [ ] 售后工单技能包上线，含完整的移动端操作流
- [ ] 3 篇真实用户案例文章（制造业老板亲自站台）
- [ ] 内容引擎产出可从 XTCRM 直接发布到至少 1 个外部渠道（公众号草稿箱）

### 第三阶段：建生态（2027 Q3-Q4）

> **目标：从"一套 CRM"变成"制造业销售管理平台"。**  
> **核心指标：技能包开发者 > 5 人，付费企业客户 > 50 家。**

#### Q3（7-9 月）：平台化

| 优先级 | 功能 | 类型 | 说明 | 预估工时 |
|--------|------|------|------|----------|
| **P0** | 技能包市场 | 平台 | 技能包在线安装/卸载/升级，支持第三方开发者提交。SkillLoader 重构支持远程下载 + 版本管理 | 4 周 |
| **P0** | 开放 API | 后端 | RESTful API + Webhook。客户创建/更新/删除事件可推送到外部系统（ERP/财务/企业微信） | 2 周 |
| **P1** | 企业微信集成 | 集成 | 企微应用：客户同步、跟进提醒推送到企微、在企微内直接写跟进。**中国制造业标配** | 3 周 |
| **P1** | 客户授信管理 | 技能包 | 基于历史回款记录，自动计算建议授信额度。超额度锁单。回款逾期自动降额 | 2 周 |
| **P1** | 多语言报价 | 技能包 | 基于 quote-cn 扩展，支持中英文双语报价、形式发票（Proforma Invoice）、装箱单 | 2 周 |
| **P2** | 钉钉集成 | 集成 | 钉钉应用：审批流打通（报价审批 → 钉钉审批 → 结果回写 CRM） | 2 周 |

#### Q4（10-12 月）：智能化

| 优先级 | 功能 | 类型 | 说明 | 预估工时 |
|--------|------|------|------|----------|
| **P0** | 销售预测引擎 | 后端 | 基于漏斗阶段 + 历史转化率 + 客户健康分，预测本月/本季度成交额。准确率目标 > 75% | 3 周 |
| **P0** | 智能排程 | 后端 | 基于客户优先级 + 地理位置 + 历史拜访频率，为销售生成每日拜访路线建议 | 2 周 |
| **P1** | 竞品情报 | 技能包 | 记录竞品动态 + 丢单竞品分析 + AI 自动生成竞争应对话术 | 2 周 |
| **P1** | 客户分级自动化 | 后端 | 基于订单金额 + 频率 + 回款 + 利润，自动调整客户等级（A/B/C/D），手动可覆盖 | 1.5 周 |
| **P2** | 行业报告 | 内容 | 每季度发布《制造业销售管理趋势报告》，基于匿名聚合数据（opt-in），反哺品牌 | 持续 |

#### 第三阶段里程碑验收标准

## 六、技能包生态矩阵

技能包是 XTCRM 的平台级能力。一个技能包 = 独立的后端 Controller/Model/Service + 独立的前端页面/路由 + 独立的数据库迁移 + 独立的 AI 工具注入。

### 6.1 技能包分类

| 类别 | 说明 | 现有 | 规划中 |
|------|------|------|--------|
| **业务类** | 扩展 CRM 业务能力 | invoice（代开发票） | after-sales（售后工单）、credit（授信管理）、competitor（竞品情报） |
| **文档类** | 生成正式商业文档 | quote-cn（中文报价） | proposal（标书方案）、multi-lang-quote（多语言报价）、packing-list（装箱单） |
| **内容类** | 营销内容生产 | content-engine（内容引擎） | social-auto（社媒自动发布）、email-campaign（邮件营销） |
| **集成类** | 连接外部系统 | — | wecom（企业微信）、dingtalk（钉钉）、erp-connector（ERP 对接） |
| **工具类** | 提升使用效率 | — | data-import（高级导入）、report-builder（自定义报表）、backup（自动备份） |

### 6.2 技能包开发优先级

## 七、技术架构演进

### 7.1 当前架构审视

```
✅ 做得好的
  • ThinkPHP 6.1 + Vue 3 前后端分离，架构清爽
  • Multi-Tenant 设计正确（全局 scope + 显式 storeId）
  • ModuleLoader + SkillLoader 双加载器，扩展性强
  • 零配置路由，开发效率高
  • SkillPipeline 动态工具注入，Agent 可扩展

⚠️ 需要改进的
  • PHP 7.4 即将 EOL，技能包上架后的兼容性风险
  • 前端 skill 文件手动复制（从 skills/ 到 src/skills/），开发体验差
  • 没有自动化测试，全靠人工验证
  • 数据库迁移没有回滚机制
  • AI 调用无鉴权/限流/缓存，高并发下成本失控
```

### 7.2 技术演进路线

#### 2026 Q3-Q4：工程化加固

| 项目 | 内容 | 优先级 |
|------|------|--------|
| **自动化测试** | PHPUnit 核心 Service 测试用例（CustomerService / InsightService / AgentService） | P0 |
| **AI 调用治理** | AiGateway 增加：请求缓存（同 query 5 分钟内复用）、速率限制、Token 用量统计 | P0 |
| **前端构建优化** | Vite 构建时自动复制 skills 文件 → 去掉手动复制步骤 | P1 |
| **错误监控** | 全局异常处理器 + 日志写入 crm_event_log（`event_type=system_error`） | P1 |
| **数据库迁移增强** | ModuleLoader 增加迁移回滚支持 + 迁移执行历史记录 | P2 |
| **CI/CD** | GitHub Actions：PHP 语法检查 + 前端 build + 自动打包 release | P2 |

#### 2027 Q1-Q2：架构升级

| 项目 | 内容 | 优先级 |
|------|------|--------|
| **PHP 8.1+ 升级** | 从 7.4 迁移到 8.1（match 语法、枚举、只读属性、Fibers），检查 ThinkPHP 6.1 兼容性 | P0 |
| **API 版本化** | `/api/v1/` vs `/api/v2/`，为开放 API 做准备 | P1 |
| **前端微前端化** | 技能包页面独立构建/部署/加载，不跟主应用耦合 | P2 |
| **数据库读写分离** | 洞察引擎扫描走只读库，业务写入走主库 | P2 |

#### 2027 Q3-Q4：平台化

| 项目 | 内容 | 优先级 |
|------|------|--------|
| **开放 API + Webhook** | RESTful API + 事件 Webhook，第三方集成的基础 | P0 |
| **插件沙箱** | 第三方技能包代码隔离，防止恶意代码 | P0 |
| **分布式任务** | 洞察引擎扫描从 Workerman Timer 迁移到独立 worker（支持大型租户） | P1 |
| **多数据库支持** | 大型租户独立数据库（当前共享数据库 + store_id 隔离不够） | P2 |

### 8.3 移动端策略

**不做原生 App。走 PWA + 响应式。**

| 理由 | 说明 |
|------|------|
| 维护成本 | 1 套代码 vs 3 套代码（iOS + Android + Web） |
| 用户场景 | 销售外勤只需要：查客户、写跟进、收提醒。PWA 完全够用 |
| 安装门槛 | 浏览器打开就能用 vs 去应用商店下载 |
| 更新速度 | 即时更新 vs 等审核 |

PWA 能力清单（按优先级）：
1. 响应式布局（写跟进、查客户）→ 2026 Q3
2. 洞察推送通知 → 2026 Q4
3. 离线写跟进（有网自动同步）→ 2027 Q1
4. 桌面图标 + 启动画面 → 2027 Q2
5. 相机拍照直接上传（跟进附件）→ 2027 Q2

## 十、风险与对策

### 10.1 市场风险

| 风险 | 可能性 | 影响 | 对策 |
|------|--------|------|------|
| **大厂杀入制造业 CRM** | 中 | 高 | 大厂做平台，我们做深度。大厂不会为五金行业定制洞察规则，我们会。**把行业壁垒堆高** |
| **AI 能力被通用 CRM 追上** | 高 | 中 | 洞察引擎的壁垒不在调 AI，在于 5 条规则背后的制造业业务 knowledge + 事件流闭环。**AI 是手段，行业 know-how 是护城河** |
| **目标用户不想付费** | 中 | 中 | 基础版永远免费。付费靠定制开发和高级功能（行业模板、高级 AI、企业微信集成）。**开源引流 → 服务变现** |
| **经济下行，制造企业缩减 IT 开支** | 低 | 中 | 开源免费是经济下行时的优势。企业砍预算，先砍 SaaS 年费，不会砍已部署的免费系统。**免费是穿越周期的铠甲** |

### 10.2 技术风险

| 风险 | 可能性 | 影响 | 对策 |
|------|--------|------|------|
| **PHP 7.4 EOF，安全漏洞无人修** | 高 | 高 | 2026 Q4 启动 PHP 8.1 升级计划，2027 Q1 完成迁移。期间锁定 PHP 补丁版本 |
| **DeepSeek API 涨价或服务中断** | 中 | 高 | AiGateway 支持多模型切换（DeepSeek / 通义千问 / 本地模型），用户自行配置 |
| **开源代码被白嫖商业化** | 中 | 低 | Apache-2.0 协议允许商用。壁垒在服务（定制开发、行业 know-how、持续迭代）。**代码可以抄，理解抄不走** |
| **多租户数据隔离被绕过** | 低 | 高 | 全局 scope + 代码审查 + 渗透测试。2026 Q4 加入自动化安全扫描 |

### 10.3 团队风险

| 风险 | 可能性 | 影响 | 对策 |
|------|--------|------|------|
| **核心开发者离职** | 中 | 高 | 代码文档完善（CLAUDE.md 已是典范）+ 架构决策记录（ADR）+ 社区贡献者梯队 |
